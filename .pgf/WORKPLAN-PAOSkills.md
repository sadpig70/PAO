# WORKPLAN-PAOSkills

> Source design: `.pgf/DESIGN-PAOSkills.md` (red-team review gate before execution)
> Mode: execute | All code, docs, and tests in English.

## POLICY

- canonical_source: PAO_plugin/{pao_runtime,scripts} and PAO_plugin/skills/lwar-runtime/schemas are the only hand-edited copies; PAO_skills bundled copies are generated by `pao build-skills` only
- plugin_untouched: PAO_plugin behavior, manifests, and skills are not modified except pao_cli gaining `build-skills`
- plugin_agnostic: no ${CLAUDE_PLUGIN_ROOT}, PAO_HOME, or `python -m pao_runtime` anywhere under PAO_skills/
- authored_vs_generated: SKILL.md and references/ are authored; pao_runtime/, scripts/, schemas/ are generated — the boundary is never crossed
- docs_sync: README, AGENTS.md, TechSpec updated in the same change set
- no_commit: no git commit/push without explicit user command
- max_verify_cycles: 2

## Execution order

1. S2_RuntimeBundling (BuildSkillsCmd → InitialBuild)

3. S3_SyncGates
4. S4_DocsSync

Node tree and criteria live in `DESIGN-PAOSkills.md`; snapshots in `status-PAOSkills.json`.
